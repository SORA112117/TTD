# TTD（Tomorrow To Do）UI設計書

> 設計方針: **Things 3ライクな清潔感** × **達成感を演出するマイクロインタラクション**
> 対話に基づく決定事項 + 2024〜2025年UXトレンドリサーチを統合

## 3. HomeView 詳細設計

### レイアウト全体像

```
┌─────────────────────────────────────┐
│ 明日の準備             ⚙️（設定へ）  │  ← NavigationBar
│─────────────────────────────────────│
│                                     │
│ 📌 今日の残タスク                    │  ← Section A（残があれば表示）
│ ┌─────────────────────────────────┐ │
│ │ ☐  会議資料を印刷               │ │
│ └─────────────────────────────────┘ │
│                                     │
│ 🌅 明日の準備  3月4日（水）          │  ← Section B ヘッダー
│ ─────────────────────────────────── │
│ 🎒 持ち物                           │  ← カテゴリ小見出し
│ ┌─────────────────────────────────┐ │
│ │ ☐  傘                           │ │
│ │ ☐  定期券                       │ │
│ │ + 追加...                       │ │  ← インライン追加
│ └─────────────────────────────────┘ │
│                                     │
│ ✅ やること                          │
│ ┌─────────────────────────────────┐ │
│ │ ☐  メール返信                   │ │
│ │ + 追加...                       │ │
│ └─────────────────────────────────┘ │
│                                     │
│ 🕘 予定                             │
│ ┌─────────────────────────────────┐ │
│ │ ☐  09:00 朝会                   │ │
│ │ + 追加...                       │ │
│ └─────────────────────────────────┘ │
│                                     │
│ 📋 準備チェック                      │
│ ┌─────────────────────────────────┐ │
│ │ + 追加...                       │ │
│ └─────────────────────────────────┘ │
│                                     │
└─────────────────────────────────────┘
```

### NavigationBar
- スタイル: `navigationBarTitleDisplayMode(.large)` → スクロールで折りたたまれる
- タイトル: 「明日の準備」
- 右ボタン: テンプレート一覧ボタン（`list.bullet.rectangle`アイコン）

### Section A: 今日の残タスクセクション
- **表示条件**: 当日分の未完了タスクが1件以上ある場合のみ表示
- ヘッダー: `📌 今日の残タスク`（`.footnote` + `.semibold` + uppercase）
- スタイル: `List(.insetGrouped)` のセクション
- 完了タスクは自動的に非表示（今日の未完了のみ）

### Section B: 明日のタスクセクション
- ヘッダー: 日付付き（例: `🌅 3月4日（水）の準備`）
- 4カテゴリをそれぞれ `.insetGrouped` セクションで表示
- 各カテゴリの末尾にインライン追加フィールドを配置

## 4. TaskRow 設計

### 通常状態

```
┌──────────────────────────────────────────┐
│  ○   タスクタイトル           09:00  │
│      （チェック）（本文 17pt）  （時刻）  │
└──────────────────────────────────────────┘
  高さ: 52pt（padding vertical 14pt）
```

| 要素 | 仕様 |
|------|------|
| チェックボックス | 22×22pt, cornerRadius 11, border 1.5pt, カラー `.indigo` |
| タスクタイトル | `.body`（17pt, Regular）、左余白 12pt |
| 時刻（任意） | `.footnote`（13pt）、`.secondary`、右寄せ |

### 完了状態

```
┌──────────────────────────────────────────┐
│  ●   タスクタイトル           09:00  │
│  （塗）（打ち消し線 + グレー）   （時刻）  │
└──────────────────────────────────────────┘
```

| 変化 | 仕様 |
|------|------|
| チェックボックス | 塗りつぶし（`.indigo`）+ `checkmark` SF Symbol |
| テキスト | `.strikethrough(true)` + `.tertiary`（透明度60%） |
| アニメーション | `.spring(response: 0.3, dampingFraction: 0.7)` |
| Haptic | `.sensoryFeedback(.success, trigger: isCompleted)` |

### スワイプ操作

```
←スワイプ（左から右へ指を動かす）
    └── 赤い削除ボタン「🗑 削除」が出現
        └── フルスワイプで即削除（allowsFullSwipe: true）

→スワイプ（右から左へ）
    └── 緑の完了ボタン「✓ 完了」
        ※ チェックボックスタップと同じ効果
```

```swift
// 実装パターン
.swipeActions(edge: .trailing, allowsFullSwipe: true) {
    Button(role: .destructive) { deleteTask(task) } label: {
        Label("削除", systemImage: "trash")
    }
}
```

## 5. インライン追加フィールド設計

各カテゴリセクション末尾に配置

```
┌──────────────────────────────────────────┐
│  ＋  明日のタスクを追加...               │
└──────────────────────────────────────────┘
```

| 要素 | 仕様 |
|------|------|
| プラスアイコン | `plus.circle.fill`、サイズ `.title3`、カラー `.indigo` |
| テキストフィールド | プレースホルダー「明日のタスクを追加...」 |
| キーボード送信 | `.submitLabel(.done)` でEnterキー = タスク追加 |
| フォーカス管理 | `@FocusState` で自動フォーカス |

**カテゴリ別プレースホルダー**:

| カテゴリ | プレースホルダー |
|----------|-----------------|
| 持ち物   | 「持っていくものを追加...」 |
| やること | 「明日やることを追加...」 |
| 予定     | 「予定を追加（時刻も入力できます）」 |
| 準備チェック | 「出発前に確認するものを追加...」 |

## 6. 全タスク完了演出

全カテゴリのタスクがすべて完了になった瞬間にトリガー

### 演出フロー

```
全タスク完了検知
    │
    ▼ 0.3秒後（アニメーション後）
コンフェッティ発射（ConfettiSwiftUI）
    │
    ▼ 同時
「準備完了！🎉」バナー表示（画面上部からスライドイン）
    │
    ▼ 2秒後
バナー自動フェードアウト
```

### バナーデザイン

```
┌─────────────────────────────────────┐
│  🎉  準備完了！明日も頑張ろう！      │
│      全 n 件 チェック完了            │
└─────────────────────────────────────┘
  背景: .indigo（ソフト）
  文字: .white
  cornerRadius: 16
  影: shadow(radius: 8)
  表示アニメ: .spring() でY軸+20ptからスライドイン
```

## 7. TemplateView 設計

```
┌─────────────────────────────────────┐
│ テンプレート          ＋（新規作成） │
│─────────────────────────────────────│
│                                     │
│ ┌──────────────────────────────────┐│
│ │ 📁  月曜の持ち物セット            ││
│ │      傘・定期券・PC               ││  ← 3件プレビュー
│ └──────────────────────────────────┘│
│                                     │
│ ┌──────────────────────────────────┐│
│ │ 📁  出張準備チェックリスト        ││
│ │      パスポート・充電器・着替え   ││
│ └──────────────────────────────────┘│
│                                     │
└─────────────────────────────────────┘
```

### テンプレートカード
- スタイル: `RoundedRectangle(cornerRadius: 12)` + `.fill(.background.secondary)`
- タイトル: `.headline`（17pt, Semibold）
- プレビュー: 最大3件をカンマ区切りで表示（`.caption`、`.secondary`）

### テンプレートからの適用
- カードを長押し → コンテキストメニュー「明日のリストに追加」
- 追加時: `withAnimation(.spring())` でリストに挿入アニメーション

## 10. カラーシステム

### セマンティックカラー定義

| トークン名 | Light | Dark | 用途 |
|------------|-------|------|------|
| `ttdBackground` | `#FFFFFF` | `#000000` | 画面背景 |
| `ttdCard` | `#F2F2F7` | `#1C1C1E` | カード・セル背景 |
| `ttdAccent` | `.indigo` | `.indigo` | チェックボックス・ボタン |
| `ttdSuccess` | `.green` | `.green` | 完了スワイプ |
| `ttdDestructive` | `.red` | `.red` | 削除スワイプ |
| `ttdPrimary` | `#000000` | `#FFFFFF` | タスクタイトル |
| `ttdSecondary` | `#6e6e73` | `#8e8e93` | サブテキスト・時刻 |
| `ttdTertiary` | `#c7c7cc` | `#48484a` | 完了済みテキスト |

> **方針**: `Color(.system〜)` や `.primary` / `.secondary` などの Adaptive Color を優先使用。
> ダークモード対応が自動化され、アクセシビリティにも強い。

## 11. タイポグラフィシステム

| 役割 | SwiftUI | サイズ | Weight | 使用箇所 |
|------|---------|--------|--------|----------|
| 画面タイトル | `.largeTitle` | 34pt | Regular | NavBar Large Title |
| セクションヘッダー | `.footnote` + `.semibold` | 13pt | Semibold | カテゴリ見出し（大文字） |
| タスクタイトル | `.body` | 17pt | Regular | タスク本文 |
| 時刻・バッジ | `.footnote` | 13pt | Regular | 予定時刻 |
| 補足テキスト | `.caption` | 12pt | Regular | テンプレートプレビュー |

### セクションヘッダーの文字装飾
```swift
Text("持ち物")
    .font(.footnote)
    .fontWeight(.semibold)
    .foregroundStyle(.secondary)
    .textCase(.uppercase)
    .kerning(0.5)
```

## 13. アニメーション設計

| インタラクション | アニメーション | 所要時間 |
|----------------|---------------|---------|
| タスク完了チェック | `.spring(response: 0.3, dampingFraction: 0.7)` + Haptic `.success` | 300ms |
| タスク追加（リスト挿入） | `.spring(response: 0.4, dampingFraction: 0.8)` | 400ms |
| タスク削除 | `.easeInOut(duration: 0.25)` | 250ms |
| 全完了バナー表示 | `.spring()` Y軸 +20pt → 0pt | 350ms |
| コンフェッティ | ConfettiSwiftUI（num: 60, radius: 350） | 3秒間 |
| 画面遷移（タブ切替） | TabViewデフォルト | — |

## 14. 使用ライブラリ

| 用途 | ライブラリ | SPM URL | バージョン |
|------|-----------|---------|-----------|
| タスク完了チェックアニメ | lottie-spm | `https://github.com/airbnb/lottie-spm.git` | 4.6.0 |
| 全完了コンフェッティ | ConfettiSwiftUI | `https://github.com/simibac/ConfettiSwiftUI.git` | 3.0.0 |

### 標準API活用（追加ライブラリ不要）
- スワイプ操作: `.swipeActions()`
- Hapticフィードバック: `.sensoryFeedback()` (iOS 17+)
- ボトムシート: `.presentationDetents()`
- リストスタイル: `.listStyle(.insetGrouped)`
- セクション間隔: `.listSectionSpacing(12)` (iOS 17+)
