# TTD（Tomorrow To Do）アプリ 設計ドキュメント

## 1. プロジェクト概要

### コンセプト
「明日の自分を助ける、今夜の5分」

**TTD（Tomorrow To Do）** は、前日夜に翌日の準備物・タスク・予定をリスト化しておき、翌朝スムーズに確認できるシンプルなiOSアプリ。
忘れ物・やり忘れをゼロにし、朝の出発をスムーズにすることが目的。

### 使用想定フロー
```
前日夜: アプリを開いて翌日のリストを入力（3〜5分）
    ↓
翌朝: プッシュ通知 → アプリを開いてリストを確認
    ↓
出発前: チェックをつけながら準備完了を確認
```

---

## 2. ターゲットユーザー

- 毎朝の忘れ物が不安な人
- 前日夜に翌日の段取りをしておきたい人
- シンプルで素早く使えるアプリを好む人
- 主な端末: iPhone（ウィジェットも活用）

---

## 3. 機能要件（MVP）

### 3.1 コアタスクカテゴリ（4種類）

| カテゴリ | 説明 | アイコン |
|----------|------|----------|
| 持ち物 | 傘・書類など物理的に持っていくもの | 🎒 |
| やること | 仕事・作業タスク | ✅ |
| 予定 | 時刻指定のあるスケジュール | 🕘 |
| 準備チェック | 出発前の確認リスト | 📋 |

### 3.2 主要機能

#### タスク管理
- タスクの追加・編集・削除
- カテゴリ別の分類
- チェックボックスで完了マーク
- 予定カテゴリは任意で時刻を設定可能

#### テンプレート機能
- リストをテンプレートとして保存（「いつもの持ち物セット」など）
- テンプレートから新しいリストを作成
- 複数テンプレートの管理

#### 通知機能
- 朝のリマインダー通知（時刻はユーザーが設定）
- 通知タップでアプリが即座に開く
- 通知内容: 「今日のリストを確認してください」

#### ウィジェット対応
- ロック画面ウィジェット: タスク数または完了進捗を表示
- ホーム画面ウィジェット（小）: 直近のタスク一覧をプレビュー

## 4. 対象外（MVPスコープ外）

以下はMVPでは実装しない（将来検討）:

- Siriショートカット連携
- 天気連携
- iPad最適化レイアウト
- 履歴アーカイブ機能（過去リストの保存）

## 5. UIデザイン方針

### デザインコンセプト
**「シンプル・ミニマル」**
白を基調とした清潔感のあるUI。朝の忙しい時間帯に素早く確認できることを最優先。

### カラーパレット（案）
```
背景:      #FFFFFF（ホワイト）
テキスト:  #1C1C1E（システムブラック）
アクセント: #007AFF（システムブルー）
完了済み:  #8E8E93（グレー・打ち消し線）
カード背景: #F2F2F7（システムグレー6）
```

### 画面構成

#### メイン画面（HomeView）
```
┌─────────────────────────┐
│  明日の準備リスト         │
│  2026/03/04 (水)        │
│─────────────────────────│
│  🎒 持ち物               │
│  ☐ 傘を持っていく         │
│  ☐ 定期券               │
│─────────────────────────│
│  ✅ やること              │
│  ☐ 会議資料を印刷         │
│─────────────────────────│
│  🕘 予定                 │
│  ☐ 09:00 朝会            │
│─────────────────────────│
│      [+ タスクを追加]     │
└─────────────────────────┘
```

#### タスク追加シート（AddTaskView）
- タイトル入力
- カテゴリ選択（セグメントまたはピッカー）
- 予定カテゴリ選択時のみ時刻ピッカーを表示

#### 設定画面（SettingsView）
- 朝の通知時刻設定
- テンプレート管理
- 翌日リセット設定（手動 or 自動）

## 6. 技術設計

### アーキテクチャ
**MVVM（Model-View-ViewModel）**

```
TTD/
├── TTDApp.swift                 // エントリポイント
├── Models/
│   ├── TaskItem.swift           // タスクのデータモデル
│   ├── TaskCategory.swift       // カテゴリ定義（enum）
│   └── TaskTemplate.swift       // テンプレートモデル
├── ViewModels/
│   ├── HomeViewModel.swift      // メイン画面のロジック
│   └── SettingsViewModel.swift  // 設定画面のロジック
├── Views/
│   ├── Home/
│   │   ├── HomeView.swift       // メイン画面
│   │   ├── TaskRowView.swift    // タスク1行のUI
│   │   └── CategorySectionView.swift // カテゴリセクション
│   ├── AddTask/
│   │   └── AddTaskView.swift    // タスク追加シート
│   ├── Settings/
│   │   └── SettingsView.swift   // 設定画面
│   └── Widget/
│       └── TTDWidget.swift      // ウィジェット
├── Services/
│   ├── NotificationService.swift // 通知管理
│   └── TemplateService.swift     // テンプレート操作
└── Shared/
    ├── Extensions/
    └── Components/              // 共通UIコンポーネント
```

### データ管理
**SwiftData（iOS 17+）**

```swift
// TaskItemモデル（概要）
@Model
class TaskItem {
    var id: UUID
    var title: String
    var category: TaskCategory
    var scheduledTime: Date?      // 予定カテゴリのみ
    var isCompleted: Bool
    var targetDate: Date          // 対象日付（翌日）
    var createdAt: Date
}

// テンプレートモデル（概要）
@Model
class TaskTemplate {
    var id: UUID
    var name: String              // 「いつもの月曜セット」など
    var tasks: [TemplateTask]     // テンプレート内のタスクリスト
    var createdAt: Date
}
```

### 通知設計
- `UNUserNotificationCenter` を使用
- 毎朝指定時刻に繰り返しローカル通知
- バッジ数 = 未完了タスク数

### ウィジェット設計
- **WidgetKit** を使用
- **小サイズ**: 未完了タスク数 + 上位2〜3件のプレビュー
- **ロック画面（アクセサリー）**: 完了率のプログレスインジケーター
